# 보안 & 개인정보 보호

MindFlow는 사용자의 개인정보를 최우선으로 보호합니다.
모든 메시지는 엔드투엔드 암호화되며, 우리는 내용을 볼 수 없습니다.

---

## ✅ 엔드투엔드 암호화 적용

**모든 1:1 대화는 종단간 암호화로 보호됩니다.**
서버에서도 메시지 내용을 볼 수 없습니다.

---

## 🔒 핵심 보안 기능

### 1. 엔드투엔드 암호화

**RSA-2048 + AES-256 하이브리드 암호화**로 메시지를 보호합니다.

- 각 사용자는 로그인 시 RSA-2048 키 쌍 생성
- 메시지는 AES-256으로 암호화
- 암호화 키는 사용자 기기에만 저장
- 서버는 암호화된 데이터만 전달

**암호화 흐름:**
```
사용자 A → [AES-256 암호화] → 서버 (암호화된 상태) → [복호화] → 사용자 B
                                ↑
                        서버는 읽을 수 없음!
```

### 2. Zero-Knowledge 아키텍처

서버는 암호화된 데이터만 전달합니다.

- **관리자를 포함한 누구도** 메시지 내용을 볼 수 없습니다
- 서버에는 복호화 키가 없습니다
- 법적 요청이 있어도 기술적으로 메시지를 복호화할 수 없습니다

### 3. 본인인증

공인 인증기관을 통한 본인인증으로 악의적인 사용자를 사전에 차단합니다.

- NICE 평가정보, KCB 등 공인 본인인증 기관 연동
- 실명, 전화번호, 생년월일 검증
- 익명성을 악용한 위법 행위 방지
- 본인인증 완료 사용자에게는 특별 뱃지 부여

### 4. 2단계 인증 (2FA)

TOTP 기반 2FA로 계정을 이중으로 보호합니다.

- Google Authenticator, Authy 등 인증 앱 지원
- 새 기기 로그인 시 이메일 알림 전송
- 비밀번호 유출 시에도 계정 보호
- 로그인 기록 확인 가능

### 5. 메시지 자동 삭제

시간 기반 자동 삭제 기능을 제공합니다.

- **1일/7일/30일** 후 자동 삭제 설정 가능
- 읽은 즉시 삭제 옵션
- 완전 삭제 시 **복구 불가능** (서버, 백업에서 모두 삭제)
- 개별 메시지 또는 전체 대화 삭제 가능

### 6. 데이터 내보내기

언제든지 모든 데이터를 다운로드할 수 있습니다.

- **JSON 형식**으로 전체 데이터 내보내기
- 메시지, 프로필, 친구 목록 등 모든 정보 포함
- 데이터는 **100% 사용자 소유**
- GDPR 제7조 데이터 이동권 준수

## 🚫 우리가 하지 않는 것

### 1. 사용자 데이터 판매

❌ **절대로 사용자 데이터를 제3자에게 판매하지 않습니다.**

- 우리의 수익 모델은 광고가 아닌 프리미엄 서비스입니다
- 사용자 데이터는 서비스 제공 목적으로만 사용됩니다
- 마케팅 목적의 데이터 판매 절대 금지

### 2. 메시지 스캐닝

❌ **메시지 내용을 스캔하거나 분석하지 않습니다.**

- 엔드투엔드 암호화로 인해 **기술적으로도 불가능**합니다
- AI 학습 데이터로 사용하지 않습니다
- 콘텐츠 필터링을 위한 스캐닝도 하지 않습니다

### 3. 광고 타겟팅

❌ **사용자 프로파일을 만들어 맞춤 광고를 제공하지 않습니다.**

- 광고 추적 쿠키 사용 안 함
- Google Analytics 사용 안 함
- 제3자 추적 스크립트 사용 안 함
- 행동 기반 광고 없음

### 4. 백도어

❌ **정부 기관을 위한 백도어나 마스터 키를 만들지 않습니다.**

- 법적 요청이 있어도 **기술적으로 메시지를 복호화할 수 없습니다**
- 암호화 우회 기능 없음
- 개인정보 보호를 최우선으로 설계
- 투명성 보고서를 통해 정부 요청 건수 공개

## 🔐 추가 보안 조치

### 전송 및 저장 보안

- **전송 중 암호화**: TLS 1.3 (HTTPS)
- **저장 시 암호화**: AES-256
- **비밀번호 암호화**: bcrypt 단방향 해시 (Salt 추가)

### 접근 제어

- IP 기반 접근 제어
- Firebase 보안 규칙을 통한 엄격한 데이터 접근 제한
- 본인의 데이터만 읽고 쓸 수 있도록 제한
- 모든 데이터 접근은 로그로 기록

### 정기 보안 감사

- **분기별** 내부 보안 점검
- **연 1회** 외부 보안 감사
- 버그 바운티 프로그램 운영 (취약점 신고 포상)
- 취약점 발견 시 즉시 공개 및 패치
