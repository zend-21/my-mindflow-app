# 보안 정책

**나의 비망록(MindFlow)**의 보안 정책입니다.

## 보안 취약점 신고

보안 취약점을 발견하신 경우 다음 방법으로 신고해 주세요:

### 신고 방법
- **이메일**: security@mindflow.app (예시)
- **GitHub Private Security Advisory**: [보안 권고 생성](https://github.com/your-repo/security/advisories/new)

### 신고 시 포함할 정보
1. 취약점의 상세 설명
2. 재현 방법 (PoC)
3. 영향 범위
4. 가능한 경우 해결 방법 제안

## 보안 업데이트 프로세스

### 1. 취약점 접수 및 확인
- 신고 접수 후 24시간 이내 확인
- 취약점 심각도 평가 (Critical, High, Medium, Low)

### 2. 패치 개발
- Critical: 24시간 이내
- High: 72시간 이내
- Medium: 1주일 이내
- Low: 다음 정기 업데이트

### 3. 배포 및 공지
- 패치 배포
- 사용자 공지 (심각도에 따라)
- CVE 할당 (필요 시)

## 보안 기능

### 인증 및 권한
- Google OAuth 2.0 사용
- JWT 토큰 기반 세션 관리
- Firebase Authentication 통합

### 데이터 보호
- 전송 중 TLS 1.3 암호화
- 저장 시 AES-256 암호화 (Firebase)
- 시크릿 문서 추가 암호화 (PIN 기반)

### 접근 제어
- Firebase Security Rules 적용
- 사용자별 데이터 격리
- 최소 권한 원칙

### 모니터링
- 비정상적인 로그인 시도 감지
- API 요청 속도 제한
- 오류 로그 수집 및 분석

## 알려진 제한사항

### 로컬 스토리지 보안
- 브라우저 로컬 스토리지는 암호화되지 않음
- 공용 컴퓨터 사용 시 주의 필요
- 시크릿 모드 사용 권장

### 제3자 의존성
- Google Firebase
- Google Drive API
- react-markdown 등 npm 패키지

## 보안 Best Practices

### 사용자를 위한 권장사항
1. **강력한 Google 계정 비밀번호 사용**
2. **2단계 인증 활성화**
3. **공용 컴퓨터에서 로그아웃 필수**
4. **의심스러운 활동 즉시 신고**

### 개발자를 위한 권장사항
1. 정기적인 의존성 업데이트
2. 코드 리뷰 필수
3. 보안 테스트 자동화
4. 최소 권한 원칙 준수

## 규정 준수

### 개인정보 보호
- GDPR 준수 (해당 시)
- 개인정보 보호법 준수 (대한민국)
- CCPA 준수 (해당 시)

### 데이터 주권
- 데이터는 Firebase 서버에 저장
- 지역별 데이터 센터 활용
- 사용자의 데이터 삭제권 보장

## 감사 로그

### 기록되는 활동
- 로그인/로그아웃
- 데이터 생성/수정/삭제
- 설정 변경
- 오류 발생

### 로그 보관 기간
- 보안 로그: 90일
- 접근 로그: 30일
- 오류 로그: 180일

## 사고 대응 절차

### 1단계: 탐지 및 평가
- 사고 확인
- 영향 범위 파악
- 심각도 평가

### 2단계: 격리 및 제거
- 위협 요소 격리
- 악성 코드/계정 제거
- 추가 피해 방지

### 3단계: 복구
- 정상 서비스 복구
- 데이터 복원 (필요 시)
- 모니터링 강화

### 4단계: 사후 분석
- 원인 분석
- 재발 방지 대책 수립
- 사용자 공지 (필요 시)
